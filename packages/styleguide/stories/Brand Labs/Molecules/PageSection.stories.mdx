You will want to use this when the main content of your page is easily divided/organized into multiple semantic sections. You would use one PageSection component to wrap each of these semantic sections. They're meant to stack vertically on the page and scroll nicely into each other.

It's suggested to utilize the optional header and footer buttons for Call To Actions that your section content might have.

By default, all of the optional buttons are rendered with the `<TextButton>` component.

## PageSection with custom Header and Footer Buttons

All of these buttons are overrideable however by passing a ReactNode as a prop instead.

<Canvas>
  <Story name="PageSection with custom Header and Footer Buttons">
    {(args) => (
      <PageSection
        {...args}
        headerButton={<IconButton icon={SearchIcon} />}
        headerSecondaryButton={<FillButton>Share</FillButton>}
        footerButton={<StrokeButton>NEXT</StrokeButton>}

        Lorem ipsum dolor sit amet, consectetur adipiscing elit. Vestibulum
