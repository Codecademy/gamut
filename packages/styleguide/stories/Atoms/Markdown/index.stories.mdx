## Element Overrides

In cases where our default Markdown renderer is not returning exactly what you need, you can override specific tags or custom tags.

<Canvas>
  <Story
    name="Element Overrides"
    args={{
      overrides: elements,
      text: `"## Hello World
          <CodeBlock>Span inside a custom code block element</CodeBlock>
          <CustomElement title='A Custom Component' />"`,
    }}
  >
    {(args) => <Markdown {...args} />}
  </Story>
</Canvas>

A common override may be to change the font size of a Heading.

<Canvas>
  <Story
    name="Tag Overrides"
    args={{ overrides: tags, text: '### Smaller H3' }}
  >
    {(args) => <Markdown {...args} />}
  </Story>
</Canvas>

If you need to override a link, iframe, or table, you also need to provide the `skipDefaultOverides` prop.

export const overrides = {
  a: {
    component: TextDeprecated,
  },
};

<Canvas>
  <Story
    name="Link Overrides"
    args={{
      text: 'This was a link](https://codecademy.com).',
      skipDefaultOverides: true,
      overrides,
    }}
  >
    <Markdown skipDefaultOverides />
  </Story>
</Canvas>

### Codepedia

<StoryStatus status="volatile" />

Used to render inline information from [Codepedia](https://codecademy.github.io/codepedia) (a hackathon project!)

<Canvas>
  <Story name="Codepedia (Standalone)">
    {(args) => (
      <Codepedia
        concept="comments"
        language="javascript"
        align="left"
        position="above"
        verticalOffset={12}
        horizontalOffset={-10}
      >
        Comments
      </Codepedia>
    )}
  </Story>
</Canvas>

<Canvas>
  <Story name="Codepedia (Override)">
    {(args) => (
      <Markdown
        text='This is an example of an embedded <Codepedia concept="comments" language="javascript">Codepedia link</Codepedia>. It will render inline for anything that uses the markdown field.'
        overrides={{
          Codepedia: {
            component: Codepedia,
            allowedAttributes: ['concept', 'language'],
          },
        }}
      />
    )}
  </Story>
</Canvas>
