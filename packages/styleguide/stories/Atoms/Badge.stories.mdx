Badges are generally used as a standard way to highlight a short piece of text, likely a single word. Badges are inline and generally found next to some interface element to be called out.

export const BadgeTemplate = (args) => (
  <Badge {...args}>{args.children ? args.children : args.variant}</Badge>
);

## Variants

Badge variants have different meanings between applications. While they are all specified on the `variant` prop not all of them can be used together.

<Canvas>
  <Story name="default" args={{ children: 'default' }}>
    {BadgeTemplate.bind({})}
  </Story>
  <Story name="stroke" args={{ variant: 'stroke' }}>
    {BadgeTemplate.bind({})}
  </Story>
  <Story name="grey" args={{ variant: 'grey' }}>
    {BadgeTemplate.bind({})}
  </Story>
  <Story name="yellow" args={{ variant: 'yellow' }}>
    {BadgeTemplate.bind({})}
  </Story>
</Canvas>

`default` and `stroke` both adapt to their <LinkTo kind='Foundations/ColorMode'>ColorMode</LinkTo> (which you can check out by using the toolbar at the top of this page!)

## Size

We have two `size` options, which should be used according to their context. Our `default` size should be used when the `Badge` is a standalone entitity, the `sm` size should be used when the `Badge` is inline with other components or `Text`.

<Canvas>
  <Story
    name="default size"
    args={{ variant: 'stroke', children: 'default size' }}
  >
    {BadgeTemplate.bind({})}
  </Story>
  <Story name="small" args={{ size: 'sm', children: 'sm size' }}>
    {BadgeTemplate.bind({})}
  </Story>
</Canvas>
