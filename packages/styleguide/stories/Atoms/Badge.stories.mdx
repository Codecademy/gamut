Badges are generally used as a standard way to highlight a short piece of text, likely a single word. Badges are inline and generally found next to some interface element to be called out.

## Usage

- To display statuses or other read-only information in your Badges
- **Do not use this prop for elements that are meant to be percieved as clickable or interactive**
- Use the component's default rectangular style when including Badges inside (or in tandem with) other dynamic elements

export const BadgeTemplate = (args) => (
  <Badge {...args}>{args.children ? args.children : args.variant}</Badge>
);

## Variants

- The `default` and `stroke` variants both adapt to their <LinkTo kind='Foundations/ColorMode'>ColorMode</LinkTo> (which you can check out by using the toolbar at the top of this page!)
- Badge variants such as `yellow`, `hyper`, and `blue` have different meanings between applications. While they are all specified on the `variant` prop, be mindful when using them along with other Badges with preexisting meanings.

## Size

We have two `size` options, which should be used according to their context:

- Use `default` when the `Badge` is a standalone entitity
- Use `sm` when the `Badge` is inline with other components such as <LinkTo kind='Typography/Text'>Text</LinkTo>
