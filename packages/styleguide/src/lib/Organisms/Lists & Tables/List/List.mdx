## Overview

`List` is the root container and it provides context to child rows and columns.

### Components

- `List`: Root list or table wrapper; which sets spacing, variant, `as` (`ul` | `ol` | `table`), scrollable layout, and container-query behavior for its children.
- `ListRow`: A single row; supports expandable content. See <LinkTo id="Organisms/Lists & Tables/List/ListRow">ListRow</LinkTo>.
- `ListCol`: Column cell with sizing, `type`, and justification. See <LinkTo id="Organisms/Lists & Tables/List/ListCol">ListCol</LinkTo>.
- `TableHeader`: Sticky header row for `as="table"`. See <LinkTo id="Organisms/Lists & Tables/List/TableHeader">TableHeader</LinkTo>.

### Design principles

Recommended for engaging with individual items grouped under a topic

- Manage and display information inside your List
- Include rich information and controls inside of each ListCol and ListRow
- Expand individual rows to surface advanced layouts

Customization with rich information

- Include icons, graphics, complex layouts, and other Atom level components

Optional List-level interactions include

- Basic filtering and sorting across common attributes across rows

Be careful about placement and positioning

- Make sure controls are visible on each row
- Place List components inside main containers to avoid overflow
- Use a table if you have a design that's meant to compare information between rows

### Usage

List components are one of the main design tools for displaying information in a structured, repeated way. They read differently from cards and behave differently from tables.

- A List component can function like a table, but isn't exclusively for displaying table data.
- A List component is distinct from a table in that it has mixed column content (interactive elements / metrics) whereas tables have only metrics. If you plan on having row controls or options in your table, use a List.
- When presenting repetitive content prefer using a List view over a Card view.

### Sample code

```tsx
import { List, ListRow, ListCol } from '@codecademy/gamut';

<List variant="default" spacing="condensed">
  <ListRow>
    <ListCol size="md">Worf</ListCol>
    <ListCol size="md">Lieutenant Commander</ListCol>
  </ListRow>
  <ListRow>
    <ListCol size="md">Deanna Troi</ListCol>
    <ListCol size="lg">Commander</ListCol>
  </ListRow>
</List>;
```

## Variants

Lists have several style variants for different use cases:

- `table`: Alternating backgrounds for visually separated rows of text.
- `default`: Bordered rows for mixed content with no vertical gutter.
- `card`: Bordered rows with a vertical gutter.
- `block`: Unbordered rows for mixed content with a vertical gutter.
- `plain`: Minimal styling with no borders, backgrounds, or special styling—just the basic list structure.

```tsx
<List variant="default" />
```

### Default

For displaying things with more abstract row contents (like buttons or custom rendered components) use the `default` variant. This structure denotes borders between rows and separates interactive elements with more clarity than tables.

<Canvas of={ListStories.Default} />

### Card

For content that doesn't need to be compared or visually adjacent to other items in a list (like current curriculum progress) use the `card` variant.

### Block

For making feature-forward designs (like the dashboard on Codecademy), or to use as page sections + scaffolding (like our syllabus pages). The `block` variant should always be used on a colored background.

For minimal, clean styling with no borders, backgrounds, or special styling, use the `plain` variant. This variant provides just the basic list structure and is ideal for simple content that doesn't need visual separation or emphasis.

## List as an ordered list (ol)

By default, `List` renders a `ul` (unordered list). All of the above configurations work with ordered lists as well. Pass `ol` to the `as` prop to render an ordered list and add numbering to the rows.

Ordered lists should always include one column with a `header` type row so numbering renders correctly.

## Advanced Features

### Layouts

#### Horizontal scrolling

If you have a list with many columns and want to avoid collapsing column content, use the `scrollable` prop. This prop keeps the row header column pinned at the start of the table and expands every column to at least its minimum flex-basis. Rows and columns scroll together along the horizontal axis so users can compare data and take actions across columns.

## Container Query Control

By default, `List` components use CSS container queries to determine responsive behavior based on the container width of the `List`, rather than the viewport width. This allows `List` components to adapt appropriately when placed in narrow sidebars or constrained layouts.

However, there are cases where you may want to disable container queries and rely on traditional media queries instead:

### When to use `disableContainerQuery={true}`

- **The `List` is in a container already smaller than the breakpoint**: When the `List` is always rendered in a narrow container (like a 300px sidebar), container queries may not behave as expected
- **The `List` has very few rows**: Short `List` components with only 1-2 rows may not benefit from container query responsive behavior
- **Custom responsive management**: When you're implementing your own media or container query logic and want to avoid conflicts
- **Performance considerations**: In rare cases where container query support may impact performance
- **Legacy browser support**: When supporting browsers that don't have full container query support

### Example usage

```tsx
// Disable container queries for a List in a narrow sidebar
<div style={{ width: '280px' }}>
  <List disableContainerQuery spacing="condensed">
    <ListRow>
      <ListCol type="header">Item</ListCol>
      <ListCol>Details</ListCol>
    </ListRow>
  </List>
</div>

// Use default container query behavior (recommended for most cases)
<List spacing="condensed">
  <ListRow>
    <ListCol type="header">Item</ListCol>
    <ListCol>Details</ListCol>
  </ListRow>
</List>
```
