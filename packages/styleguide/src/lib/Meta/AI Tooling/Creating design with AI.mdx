## Figma Make

**Designers:** Use the **published Gamut Make kit** in Figma when building in Make — add it to your Make file from the org kit picker. Kits are configured entirely in the Figma app, not via a local CLI.

**How kits work:** [Get started with Make kits](https://arc.net/l/quote/salxxuaw) (canonical: [Figma Help](https://help.figma.com/hc/en-us/articles/39241689698839)).

**Maintainers:** The [`guidelines/`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/guidelines/) tree in `packages/gamut/agent-tools` is source markdown for the published kit. Ship updates through the kit in Figma (upload or edit guidelines in the Make file’s code explorer). See [Add guidelines to Figma Make](https://help.figma.com/hc/en-us/articles/33665861260823).

## Gamut plugin (Cursor and Claude)

Install Gamut agent tools into your editor with the `gamut` CLI from the `@codecademy/gamut` package:

```sh
# Cursor (default) — skills, rules, commands, agents, guidelines
gamut plugin install
gamut plugin install cursor --theme core

# Claude Code + product DESIGN.md at repo root
gamut plugin install claude --theme percipio

# Check status
gamut plugin list

# Refresh after gamut updates
gamut plugin update
gamut plugin update cursor --theme core --force
```

**What gets installed:** Content from [`packages/gamut/agent-tools/`](https://github.com/Codecademy/gamut/tree/main/packages/gamut/agent-tools) — `skills/`, `rules/`, `commands/`, `agents/`, and `guidelines/` (implementation reference; same markdown is also published to the Figma Make kit).

**Related agent skills** in the repo include `gamut-system-props`, `gamut-color-mode`, `gamut-style-utilities`, `gamut-theming`, `gamut-testing`, and others under `packages/gamut/agent-tools/skills/`.

## DESIGN.md — product design context for AI agents

Gamut ships **YAML design-context files** in [`packages/gamut/agent-tools/`](https://github.com/Codecademy/gamut/tree/main/packages/gamut/agent-tools) (not under `guidelines/`). They encode palette hex, semantic color aliases, typography and spacing notes, and component usage patterns so agents and [`gamut-review`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/commands/gamut-review.md) align with the right product.

| Source file in gamut                                                                                                    | Product           | Notes                                                                   |
| ----------------------------------------------------------------------------------------------------------------------- | ----------------- | ----------------------------------------------------------------------- |
| [`DESIGN.Codecademy.md`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/DESIGN.Codecademy.md) | Codecademy (Core) | Primary reference; YAML with `colors`, semantic aliases, patterns       |
| [`DESIGN.Percipio.md`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/DESIGN.Percipio.md)     | Percipio          | Percipio-specific tokens and semantics                                  |
| [`DESIGN.LXStudio.md`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/DESIGN.LXStudio.md)     | LX Studio         | LX Studio–specific tokens                                               |
| [`DESIGN.md`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/DESIGN.md)                       | —                 | **Deprecated** stub — do not copy; use the matching `DESIGN.*.md` above |

### Setup in your app repo

Run from the **repository root**:

```sh
gamut plugin install cursor --theme core
# or: percipio | lxstudio | admin | platform (admin/platform use Codecademy DESIGN)
```

`--theme` installs the plugin **and** copies the matching `DESIGN.*.md` to **`DESIGN.md`** at the repo root. Use `--force` to overwrite an existing `DESIGN.md`.

**Manual fallback:** copy `DESIGN.Codecademy.md`, `DESIGN.Percipio.md`, or `DESIGN.LXStudio.md` from agent-tools to the repo root and rename to `DESIGN.md`.

**Plugin only (no DESIGN.md):** `gamut plugin install cursor` — installs `skills`, `rules`, `commands`, `agents`, and `guidelines` only. [`gamut-review`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/commands/gamut-review.md) will flag a missing `DESIGN.md` and suggest `--theme`.

### How this differs from other AI context

| Artifact                         | Where it lives                                                               | Used for                                                                               |
| -------------------------------- | ---------------------------------------------------------------------------- | -------------------------------------------------------------------------------------- |
| `DESIGN.md` (from `DESIGN.*.md`) | App repo root                                                                | Coding agents + `gamut-review` — product tokens, semantic roles, patterns              |
| `guidelines/` in agent-tools     | Cursor plugin after `gamut plugin install`; also published to Figma Make kit | Implementation reference (setup, foundations, components) — skills and rules link here |
| `skills/`, `rules/`              | Cursor/Claude after `gamut plugin install`                                   | Task playbooks and always-on guardrails for Gamut code                                 |

For semantic tokens in React components, see [Best practices](https://gamut.codecademy.com/?path=/docs-meta-best-practices--page). For palette verification against DESIGN YAML, see [`guidelines/foundations/color.md`](https://github.com/Codecademy/gamut/blob/main/packages/gamut/agent-tools/guidelines/foundations/color.md) in the agent-tools package.
