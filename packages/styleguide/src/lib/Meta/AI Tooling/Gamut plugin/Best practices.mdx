## Recommended setup

From your **app repo root**, install the plugin with the product theme that matches your app:

```sh
gamut plugin install cursor --theme core
```

Use `percipio`, `lxstudio`, `admin`, or `platform` when that matches your product. Refresh after upgrading `@codecademy/gamut`:

```sh
gamut plugin update cursor --theme core --force
```

## Audit with gamut-review

Before large PRs or when onboarding an existing codebase, use the **`gamut-review`** skill. It checks for `DESIGN.md`, Gamut dependencies, `GamutProvider`, import patterns, hardcoded hex colors, and test conventions.

Without `DESIGN.md` at the repo root, color findings are lower confidence ‚Äî install with `--theme` or copy the matching `DESIGN.*.md` manually.

## Agent skills

Related skills under [`packages/gamut/agent-tools/skills/`](https://github.com/Codecademy/gamut/tree/main/packages/gamut/agent-tools/skills/):

- `gamut-review` ‚Äî codebase audit playbook
- `gamut-theming` ‚Äî theme selection, `GamutProvider`, `theme.d.ts`
- `gamut-color-mode` ‚Äî ColorMode and semantic color
- `gamut-buttons` ‚Äî button atoms, variants, disabled patterns
- `gamut-list` ‚Äî List, ListRow, ListCol primitives for custom layouts
- `gamut-datatable` ‚Äî DataTable for sortable/filterable bulk data comparison
- `gamut-datalist` ‚Äî DataList for item management with expansion and selection
- `gamut-layout` ‚Äî spacing scale, breakpoints, page grid
- `gamut-system-props` ‚Äî `system.*` / `Box`
- `gamut-style-utilities` ‚Äî `css`, `variant`, `states`
- `gamut-typography`, `gamut-forms`, `gamut-accessibility`, `gamut-testing`
- `gamut-create-skill` ‚Äî workflow playbook for authoring a new Gamut agent skill from scratch
