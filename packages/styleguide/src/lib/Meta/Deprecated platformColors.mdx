The deprecated `platformColors` object in `@codecademy/gamut-styles` is still exported for backwards compatibility. Prefer <LinkTo id="Foundations/Theme/PlatformTheme">Platform Theme</LinkTo> `theme.colors` and the active `platformPalette` in `colors.ts` for new work.

The page is split into two tables: **Used in Monaco** lists only the deprecated purple stops still referenced by the editor theme (<Code>purple[500]</Code>, <Code>purple[600]</Code>, <Code>purple[800]</Code>, <Code>purple[900]</Code>). **Not used in Monaco** lists every other legacy <Code>platformColors</Code> swatch (for example the mint ramp and lighter purples) that Monaco does not consume.

Each row uses the legacy swatch id (<Code>family-weight</Code>) with the closest opaque <Code>theme.colors</Code> entry by **ΔE_OK** in OK color space (same method as <LinkTo id="Meta/Deprecated editorColors">Deprecated editorColors</LinkTo>). Matching uses the resolved hex map (<Code>corePalette</Code> plus <Code>platformPalette</Code>). Only <Code>#RRGGBB</Code> values are considered; Monaco chrome keys are excluded from the candidate set.
