## Usage

Use SelectDropdown to pick from exclusive options within a styled dropdown.

`SelectDropdown` has a larger JavaScript dependency size than our standard `Select` component so should not be used unless specified in designs. These dropdowns are optionally searchable through the `isSearchable` prop.

If you are using this in a standard form, `SelectDropdown` must be provided an `aria-label` and a `name`. The `aria-label` must match the `htmlFor` of the FormGroupLabel or `<label>` of the SelectDropdown.

## Events and controlled components

Instead of a standard HTML Select event return, SelectDropdown returns the selected option. This is formatted as `{label: 'labelText', value: 'valueText' }`. Your event functions should be formatted as `(event) => changeHandler(event.value)` instead of `(event) => changeHandler(event.target.value)`.

SelectDropdown does not take a defaultValue prop. If you wish to define a default value, you must format `SelectDropdown` as a [controlled component](https://reactjs.org/docs/forms.html#controlled-components), like so:

```tsx
import { Box, SelectDropdown } from '@codecademy/gamut';
import React, { useState } from 'react';

export const CustomDropdown: React.FC = ({
  errorMessage,
  disabled,
  defaultValue,
  options,
}) => {
  const [currentValue, setCurrentValue] = useState(defaultValue);

  const changeHandler = (newValue) => {
    setCurrentValue(newValue);
  };

  return (
    <FormGroup error={errorMessage} disabled={disabled}>
      <SelectDropdown
        options={options}
        onChange={(e) => changeHandler(e.value)}
        placeholder="Placeholder"
        error={!!errorMessage}
        disabled={disabled}
        value={currentValue}
      />
    </FormGroup>
  );
};
```

### Abbreviated options

For space-constrained layouts or when you need to display long option names in a compact way, you can use the abbreviated feature. This allows you to show a short abbreviation in the input field while displaying the full label in the dropdown.

#### How it works

- **Input Display**: Shows the abbreviated text (e.g., "USA")
- **Dropdown Display**: Shows the full label (e.g., "United States of America")
- **Implementation**: Add an `abbreviation` property to your option objects

### Independent width control

You can control the width of the input field and dropdown independently using the `inputWidth` and `dropdownWidth` props. This is particularly useful when you need a compact input but want to show more detailed information in the dropdown.

#### How it works

- **`inputWidth`**: Controls the width of the input field (can be a string like "150px" or a number)
- **`dropdownWidth`**: Controls the width of the dropdown menu (can be a string like "350px" or a number)

#### Small size with independent widths

### Menu alignment

When using different widths for the input and dropdown, you can control how the dropdown aligns relative to the input using the `menuAlignment` prop.

#### How it works

- **`menuAlignment`**: Controls the horizontal alignment of the dropdown menu relative to the input
  - **`'left'`** (default): Aligns the dropdown to the left edge of the input
  - **`'right'`**: Aligns the dropdown to the right edge of the input
- **Use cases**: Useful when the dropdown is wider or narrower than the input, or when you need to prevent the dropdown from extending beyond the viewport
