## Usage

Use `SelectDropdown` when designs call for a styled dropdown menu, searchable options, multi-select tags, creatable options, or rich option content (icons, subtitles, grouping). It carries a larger JavaScript dependency than the standard `Select` component (react-select), so prefer `Select` for simple single-select fields where that extra weight isn't justified by the design.

When used inside a form, provide both a `name` prop and an `aria-label` that matches the `htmlFor` of the associated `FormGroupLabel` or `<label>`.

## Multiple selection

Set `multiple` to let users pick more than one option. Selected values render as removable tags in the input, and `onChange` receives the full array of selected option objects rather than a single value.

Single-select is always controlled (`value` string required). Multi-select supports both modes:

<ControlledModeTable />

### Uncontrolled

Omit `value` and the component tracks selection internally. Use this when no other part of the UI needs to react to the current selection — for example, a tags field where you only read the value at form submission.

<Canvas of={SelectDropdownStories.MultipleSelect} />

### Controlled

Pass `value` as a `string[]` and update it in `onChange`. Use this when you need to pre-populate from an API, reset the field on cancel, sync with react-hook-form, or show the selection elsewhere in the UI.

```tsx
const [value, setValue] = useState<string[]>(['Apple']);

<SelectDropdown
  multiple
  name="tags"
  options={options}
  value={value}
  onChange={(selected) => setValue(selected.map((option) => option.value))}
/>;
```

<Canvas of={SelectDropdownStories.MultipleSelectControlled} />

`onChange` receives option objects (`{ label, value }`), not `event.target.value`. The second argument is react-select action metadata — use `meta.action === 'create-option'` in creatable mode.

## Creatable

Set `isCreatable` to let users type and add options not already in the list. When the input doesn't match an existing option, an **Add "value"** row appears; selecting it fires `onCreateOption` with the typed string. The consumer owns persistence — append the new value to the `options` array.

Creatable forces `isSearchable: true` since creation requires a text input. Passing `isSearchable={false}` alongside `isCreatable` is a TypeScript error.

### Multiple selection — uncontrolled

Combine `isCreatable` with `multiple` and omit `value`. New options are appended via `onCreateOption`; existing selection is kept internally.

<Canvas of={SelectDropdownStories.CreatableMultiUncontrolled} />

### Multiple selection — controlled

Pass `value` as `string[]` and update it in `onChange` on every change, including `create-option`. Updating `options` alone in `onCreateOption` without also syncing `value` in `onChange` will clear the current selection when `options` re-renders.

<Canvas of={SelectDropdownStories.CreatableMulti} />

### Validation

Gate which inputs can be created with `isValidNewOption` — return `false` to hide the **Add** row. Use `validationMessage` to replace the default "No options" text with contextual feedback, and mirror the same message in `FormGroup`'s `error` prop so it's surfaced below the field.

Because react-select clears the input on blur, persist validation state via `onInputChange`: validate on `input-change`, and re-validate from the last typed value on `input-blur` so the `FormGroup` error isn't lost when the input clears.

## Custom input props

Pass an `inputProps` object with `hidden` and `combobox` keys to forward attributes directly onto the underlying input elements — for example, `data-*` and `aria-*` attributes for testing or analytics.

- `hidden`: forwarded to the hidden input used for form submission
- `combobox`: forwarded to the visible react-select input
