## Usage

Use SelectDropdown to pick from exclusive options within a styled dropdown.

`SelectDropdown` has a larger JavaScript dependency size than our standard `Select` component so should not be used unless specified in designs. These dropdowns are optionally searchable through the `isSearchable` prop.

If you are using this in a standard form, `SelectDropdown` must be provided an `aria-label` and a `name`. The `aria-label` must match the `htmlFor` of the FormGroupLabel or `<label>` of the SelectDropdown.

## Multiple selection

Set `multiple` to let users choose more than one option. Selected options render as removable tags in the input, and `onChange` receives the full array of selected options (each as `{ label, value }`) rather than a single value. See [Controlled vs. Uncontrolled](#controlled-vs-uncontrolled) for when to pass `value` as a `string[]`.

## Custom input props

## Abbreviated options

For space-constrained layouts or when you need to display long option names in a compact way, you can use the abbreviated feature. This allows you to show a short abbreviation in the input field while displaying the full label in the dropdown.

### How it works

- **Input Display**: Shows the abbreviated text (e.g., "USA")
- **Dropdown Display**: Shows the full label (e.g., "United States of America")
- **Implementation**: Add an `abbreviation` property to your option objects

## Independent width control

You can control the width of the input field and dropdown independently using the `inputWidth` and `dropdownWidth` props. This is particularly useful when you need a compact input but want to show more detailed information in the dropdown.

### How it works

- **`inputWidth`**: Controls the width of the input field (can be a string like "150px" or a number)
- **`dropdownWidth`**: Controls the width of the dropdown menu (can be a string like "350px" or a number)

## Menu alignment

When using different widths for the input and dropdown, you can control how the dropdown aligns relative to the input using the `menuAlignment` prop.

### How it works

- **`menuAlignment`**: Controls the horizontal alignment of the dropdown menu relative to the input
  - **`'left'`** (default): Aligns the dropdown to the left edge of the input
  - **`'right'`**: Aligns the dropdown to the right edge of the input
- **Use cases**: Useful when the dropdown is wider or narrower than the input, or when you need to prevent the dropdown from extending beyond the viewport

## Creatable

Set `isCreatable` to let users add options that aren't already in the list. When the typed value doesn't match an existing option, an **Add "value"** row appears at the bottom of the menu; selecting it fires `onCreateOption` with the typed string. The consumer owns persistence — append the new value to your `options` array (and, for multi-select, it is added to the current selection).

Creatable mode is **searchable by default** — `isSearchable` is forced to `true`, since creation needs a text input. Passing `isSearchable={false}` alongside `isCreatable` is a TypeScript error.

### Basic usage

A single-select creatable dropdown. Type a value that isn't listed, choose the **Add** row, and `onCreateOption` appends it to the options.

<Canvas of={SelectDropdownStories.Creatable} />

### Multiple selection (uncontrolled)

Combine `isCreatable` with `multiple` and omit `value`. Use `onCreateOption` to persist new options; selection is managed internally. See [Controlled vs. Uncontrolled](#controlled-vs-uncontrolled) for the full pattern.

<Canvas of={SelectDropdownStories.CreatableMultiUncontrolled} />

### Multiple selection (controlled)

Pass `value` as `string[]` and update it in `onChange` on every change, including `create-option`. See [Controlled vs. Uncontrolled](#controlled-vs-uncontrolled) for the full pattern.

<Canvas of={SelectDropdownStories.CreatableMulti} />

### Validation and inline messages

Gate which inputs can be added with `isValidNewOption` — return `false` to hide the **Add** row (e.g. minimum length, pattern matching, or de-duplication). Use `validationMessage` to replace the menu's default "No options" text with contextual feedback, and mirror the same message in a `FormGroup` `error` so it's also surfaced below the field.

When persisting a `FormGroup` error after blur, handle `onInputChange` in the consumer — react-select clears the input on blur by default. Update validation on `input-change`, and on `input-blur` re-validate from the last typed value so the error isn't lost when the input clears.

## Controlled vs. Uncontrolled

SelectDropdown does not accept a `defaultValue` prop. How you manage selection depends on single vs. multi mode:

<ControlledModeTable />

`onChange` receives the selected option as `{ label, value }` (or an array of those objects in multi mode), not `event.target.value`. The second argument is react-select action metadata — for creatable creates, `meta.action === 'create-option'`.

### Single select (controlled)

```tsx
import { FormGroup, SelectDropdown } from '@codecademy/gamut';
import { useState } from 'react';

export const SingleSelectExample = ({ options, defaultValue }) => {
  const [value, setValue] = useState(defaultValue);

  return (
    <FormGroup htmlFor="country" isSoloField label="Country">
      <SelectDropdown
        name="country"
        options={options}
        value={value}
        onChange={(option) => setValue(option.value)}
      />
    </FormGroup>
  );
};
```

### Multi select (uncontrolled)

Omit `value` and the component keeps selection in internal state. Pass `onChange` only if you need to react to changes.

```tsx
<SelectDropdown
  multiple
  name="tags"
  options={['Apple', 'Banana', 'Cherry']}
  onChange={(selected) => console.log(selected.map((o) => o.value))}
/>
```

<Canvas of={SelectDropdownStories.MultipleSelect} />

### Multi select (controlled)

```tsx
const [value, setValue] = useState<string[]>(['Apple']);

<SelectDropdown
  multiple
  name="tags"
  options={options}
  value={value}
  onChange={(selected) => setValue(selected.map((option) => option.value))}
/>;
```

Creatable modes follow the same single/multi rules above — see the [Creatable](#creatable) section for runnable examples.
