## Usage

Use a tag to categorize content, represent user inputs, or guide users to related actions. Tags can either be dismissable individually, or read-only when paired with other controls.

### Best practices:

- Choose the tag variant that aligns most with your use case.
- Allow tags to wrap to a second line when they exceed the available space, rather than truncating them.
- When displaying multiple tags, arrange them in a row and allow overflow to wrap to the next line.

### When NOT to use:

- Primary actions - for performing critical or primary actions, use the <LinkTo id="Atoms/Buttons">Button</LinkTo> component instead.
- Prominent navigation - for standalone, high-priority navigation (e.g., links to key pages or sections), use an <LinkTo id="Typography/Anchor">Anchor</LinkTo> link instead.
- Emphasizing statuses or attributes - for highlighting statuses or attributes, use the <LinkTo id="Atoms/Badge">Badge</LinkTo> component instead.

## Variants

Tags have different usage variants (`readOnly`, `selection`, `navigation`, and `suggestion`) that also come in different size variants (`default` and `large`). Aside from `readOnly`, all other Tag variants can be disabled.
