## Usage

Use a tag to categorize content, represent user inputs, or guide users to related actions. Tags can either be dismissable individually, or read-only when paired with other controls.

### Best practices:

- Choose the tag variant that aligns most with your use case.
- Allow tags to wrap to a second line when they exceed the available space, rather than truncating them.
- When displaying multiple tags, arrange them in a row and allow overflow to wrap to the next line.

### When NOT to use:

- Primary actions - for performing critical or primary actions, use the <LinkTo id="Atoms/Buttons">Button</LinkTo> component instead.
- Prominent navigation - for standalone, high-priority navigation (e.g., links to key pages or sections), use an <LinkTo id="Typography/Anchor">Anchor</LinkTo> link instead.
- Emphasizing statuses or attributes - for highlighting statuses or attributes, use the <LinkTo id="Atoms/Badge">Badge</LinkTo> component instead.

1. **Leading icon** _(optional)_

- Use to further clarify the tag meaning

2. **Badge label**

- Limit to 1–2 keywords that help to label, categorize, or organize
- _Exception_: For the Suggestion variant, can be a short phrase
- Do not truncate text to keep the information easily scannable and understandable

3. **Remove button** _(selection tags only)_

- Allows users to remove a selection

## Variants

Tags have different usage variants (`readOnly`, `selection`, `navigation`, and `suggestion`) that also come in different size variants (`default` and `large`). Aside from `readOnly`, all other Tag variants can be disabled.

### Read-only

Use the Read-only tag to categorize or label content in a static, non-interactive way. This tag is ideal for simple categorization that doesn’t require user interaction.

<Canvas of={TagStories.ReadOnly} />

### Selection

Use the Selection tag to represent specific pieces of information entered by a user. This tag is dismissible, allowing users to manage or remove their entries as needed.

<Canvas of={TagStories.Selection} />

### Navigation

Use the Navigation tag to guide users to more information by linking to related content on the tagged item. This tag is interactive and intended to provide users with quick access to additional details.

<Canvas of={TagStories.Navigation} />

### Suggestion

Use the Suggestion tag to help narrow user intent by offering dynamic suggestions, such as possible responses or actions. This tag adapts based on user interactions, supporting decision-making by presenting context-relevant options.

<Canvas of={TagStories.Suggestion} />
