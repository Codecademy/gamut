Add a semantic z-index scale (`zIndexes`) and migrate Gamut off ad-hoc z-index values.

**gamut-styles**

- Add the `zIndexes` token scale (exported object + theme scale with `--zIndexes-*` CSS variables):
  `underlay (-100)`, `base (0)`, `foreground (100)`, `portal (200)`, `widget (300)`, `appBar (400)`,
  `flyout (500)`, `modal (600)`, `popover (700)`, `toaster (800)`, `tooltip (900)`.
- It is a _soft_ scale: the `zIndex` prop stays numeric, so tokens are used as `zIndex={zIndexes.modal}`
  and raw-number escape hatches / arithmetic (`zIndexes.foreground - 2`) still work.
- `elements.headerZ` now aliases `zIndexes.appBar` (was `15`). Prefer `zIndexes.appBar` going forward.

**eslint-plugin-gamut**

- Add rule `gamut/no-raw-z-index`, enabled as `error` in the recommended config. It flags raw numeric
  z-index literals in style objects and JSX props and steers to `zIndexes` tokens; token references and
  arithmetic are allowed, and deliberate escape hatches use an inline `eslint-disable`. Major because
  the new error-level recommended rule can fail existing lint in consumers until raw z-index is migrated.
